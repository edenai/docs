Tool calling (also known as function calling) lets LLMs invoke functions you define. Instead of generating a text answer, the model returns a structured request to call a specific function with extracted arguments. Your application executes the function and sends the result back to the model for a final response.

## How It Works

1. You define **tools** (functions with names, descriptions, and parameter schemas) in your request.
2. The model analyzes the user's message and decides whether to call a tool.
3. If a tool call is needed, the model returns a `tool_calls` response instead of plain text.
4. Your code executes the function and sends the result back as a `tool` message.
5. The model uses the function result to generate its final answer.

## Supported Providers

Tool calling is supported by models from these providers:

| Provider | Models |
|----------|--------|
| OpenAI | GPT-4, GPT-4o, GPT-4o-mini, GPT-4 Turbo |
| Anthropic | Claude 3+, Claude 3.5+, Claude 4 |
| Google | Gemini 1.5+, Gemini 2.5 |
| Mistral | Mistral Large, Mistral Medium |

<Note>
Tool calling also works with [smart routing](/v3/llms/smart-routing) (`model: "@edenai"`). The router automatically selects a model that supports tools when tools are included in the request.
</Note>

## Defining Tools

Tools follow the OpenAI function calling format. Each tool has a `type` of `"function"` and a `function` object with `name`, `description`, and `parameters` (using JSON Schema):

```json
{
  "type": "function",
  "function": {
    "name": "get_weather",
    "description": "Get the current weather for a given location",
    "parameters": {
      "type": "object",
      "properties": {
        "location": {
          "type": "string",
          "description": "The city name, e.g. 'Paris' or 'New York'"
        },
        "unit": {
          "type": "string",
          "enum": ["celsius", "fahrenheit"],
          "description": "Temperature unit"
        }
      },
      "required": ["location"]
    }
  }
}
```

<Tip>
Write clear, specific descriptions for your tools and parameters. The model uses these descriptions to decide when and how to call each function.
</Tip>

## Full Example

This example walks through the complete tool calling flow: sending a request with tools, handling the tool call, and returning the result.

### Step 1: Send a Request with Tools

<CodeGroup>
```python Python
import requests
import json

url = "https://api.edenai.run/v3/chat/completions"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

# Define available tools
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get the current weather for a given location",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "The city name, e.g. 'Paris'"
                    },
                    "unit": {
                        "type": "string",
                        "enum": ["celsius", "fahrenheit"],
                        "description": "Temperature unit (default: celsius)"
                    }
                },
                "required": ["location"]
            }
        }
    }
]

payload = {
    "model": "openai/gpt-4o",
    "messages": [
        {"role": "user", "content": "What's the weather like in Paris?"}
    ],
    "tools": tools
}

response = requests.post(url, headers=headers, json=payload)
result = response.json()

# Check if the model wants to call a tool
message = result["choices"][0]["message"]
print(json.dumps(message, indent=2))
```

</CodeGroup>

The model responds with a `tool_calls` array instead of text content:

```json
{
  "role": "assistant",
  "content": null,
  "tool_calls": [
    {
      "id": "call_abc123",
      "type": "function",
      "function": {
        "name": "get_weather",
        "arguments": "{\"location\": \"Paris\", \"unit\": \"celsius\"}"
      }
    }
  ]
}
```

<Info>
When the model decides to call a tool, `finish_reason` is set to `"tool_calls"` and `content` is typically `null`.
</Info>

### Step 2: Execute the Function and Send Results Back

{/* skip-test */}
<CodeGroup>
```python Python
import requests
import json

url = "https://api.edenai.run/v3/chat/completions"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

# Your actual function implementation
def get_weather(location: str, unit: str = "celsius") -> str:
    """Call your weather API here. This is a mock example."""
    return json.dumps({
        "location": location,
        "temperature": 22,
        "unit": unit,
        "condition": "Partly cloudy"
    })

# Assume we got the tool_calls response from Step 1
tool_call = {
    "id": "call_abc123",
    "type": "function",
    "function": {
        "name": "get_weather",
        "arguments": "{\"location\": \"Paris\", \"unit\": \"celsius\"}"
    }
}

# Execute the function
args = json.loads(tool_call["function"]["arguments"])
function_result = get_weather(**args)

# Send the result back to the model
payload = {
    "model": "openai/gpt-4o",
    "messages": [
        {"role": "user", "content": "What's the weather like in Paris?"},
        {
            "role": "assistant",
            "content": None,
            "tool_calls": [tool_call]
        },
        {
            "role": "tool",
            "tool_call_id": tool_call["id"],
            "content": function_result
        }
    ],
    "tools": [
        {
            "type": "function",
            "function": {
                "name": "get_weather",
                "description": "Get the current weather for a given location",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "location": {"type": "string", "description": "The city name"},
                        "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
                    },
                    "required": ["location"]
                }
            }
        }
    ]
}

response = requests.post(url, headers=headers, json=payload)
result = response.json()

# The model now generates a natural language response using the function result
print(result["choices"][0]["message"]["content"])
# Output: "The weather in Paris is currently 22°C and partly cloudy."
```
</CodeGroup>

## Multiple Tools

You can define multiple tools in a single request. The model decides which tool(s) to call based on the user's message:

{/* skip-test */}
<CodeGroup>
```python Python
import requests
import json

url = "https://api.edenai.run/v3/chat/completions"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get current weather for a location",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {"type": "string", "description": "City name"}
                },
                "required": ["location"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "get_stock_price",
            "description": "Get the current stock price for a ticker symbol",
            "parameters": {
                "type": "object",
                "properties": {
                    "ticker": {"type": "string", "description": "Stock ticker symbol, e.g. AAPL"}
                },
                "required": ["ticker"]
            }
        }
    }
]

payload = {
    "model": "openai/gpt-4o",
    "messages": [
        {"role": "user", "content": "What's the weather in London and the current price of AAPL?"}
    ],
    "tools": tools
}

response = requests.post(url, headers=headers, json=payload)
result = response.json()

# The model may return multiple tool_calls
tool_calls = result["choices"][0]["message"].get("tool_calls", [])
for tc in tool_calls:
    print(f"Function: {tc['function']['name']}")
    print(f"Arguments: {tc['function']['arguments']}")
    print()
```
</CodeGroup>

## Using Tools with Smart Routing

Tools work seamlessly with smart routing. The router considers tool compatibility when selecting a model:

<CodeGroup>
```python Python
import requests

url = "https://api.edenai.run/v3/chat/completions"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

tools = [
    {
        "type": "function",
        "function": {
            "name": "search_database",
            "description": "Search the product database",
            "parameters": {
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "Search query"}
                },
                "required": ["query"]
            }
        }
    }
]

payload = {
    "model": "@edenai",
    "router_candidates": [
        "openai/gpt-4o",
        "anthropic/claude-sonnet-4-5",
        "google/gemini-2.5-flash"
    ],
    "messages": [
        {"role": "user", "content": "Find me wireless headphones under $100"}
    ],
    "tools": tools
}

response = requests.post(url, headers=headers, json=payload)
result = response.json()

print(f"Model used: {result['model']}")
message = result["choices"][0]["message"]
if message.get("tool_calls"):
    print(f"Tool called: {message['tool_calls'][0]['function']['name']}")
```
</CodeGroup>

## Best Practices

- **Write descriptive function names and descriptions.** The model relies on these to decide when to call a tool.
- **Keep parameter schemas simple.** Use clear property names and add `description` fields.
- **Always validate function arguments.** The model may produce unexpected values.
- **Handle the case where the model does not call a tool.** Sometimes the model answers directly without using tools.
- **Include tools in follow-up messages.** When sending tool results back, include the original `tools` array so the model can make additional calls if needed.
