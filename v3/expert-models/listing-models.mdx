---
title: "Listing Models"
---

Use the `/v3/info` endpoints to discover available features, providers, and models programmatically. This is useful for building dynamic UIs, validating model strings, or exploring what Eden AI offers.

## Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /v3/info` | List all available features and subfeatures |
| `GET /v3/info/{feature}/{subfeature}` | Get details for a specific feature (providers, models, pricing, schemas) |

## List All Features

Retrieve every feature and subfeature available on the Universal AI endpoint.

<CodeGroup>
```bash cURL
curl -X GET https://api.edenai.run/v3/info \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}

response = requests.get(
    "https://api.edenai.run/v3/info",
    headers=headers
)

features = response.json()
for feature in features:
    print(feature)
```

```javascript JavaScript
const response = await fetch("https://api.edenai.run/v3/info", {
  headers: {
    "Authorization": "Bearer YOUR_API_KEY"
  }
});

const features = await response.json();
console.log(features);
```
</CodeGroup>

## Get Feature Details

Retrieve detailed information about a specific feature, including available providers, models, pricing, and input/output schemas.

<CodeGroup>
```bash cURL
curl -X GET https://api.edenai.run/v3/info/text/moderation \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```python Python
import requests

headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}

response = requests.get(
    "https://api.edenai.run/v3/info/text/moderation",
    headers=headers
)

details = response.json()

# List available providers
for provider in details.get("providers", []):
    print(f"Provider: {provider['name']}")
    if "models" in provider:
        for model in provider["models"]:
            print(f"  Model: {model}")
```

```javascript JavaScript
const response = await fetch("https://api.edenai.run/v3/info/text/moderation", {
  headers: {
    "Authorization": "Bearer YOUR_API_KEY"
  }
});

const details = await response.json();
console.log(JSON.stringify(details, null, 2));
```
</CodeGroup>

## Response Contents

The feature details response includes:

| Field | Description |
|-------|-------------|
| `providers` | List of available providers and their supported models |
| `pricing` | Cost information per provider |
| `input_schema` | JSON Schema describing the expected input format |
| `output_schema` | JSON Schema describing the response format |

<Tip>
Use the `input_schema` and `output_schema` fields to understand exactly what parameters each feature accepts and what the response looks like. This is especially helpful when integrating a feature for the first time.
</Tip>

## Use Cases

- **Dynamic model selection** — build UIs that let users pick providers from the live list.
- **Validation** — verify a model string is valid before sending a request.
- **Cost comparison** — compare pricing across providers for the same feature.
- **Schema inspection** — check input requirements without reading documentation.
