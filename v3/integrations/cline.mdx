Configure [Cline](https://github.com/cline/cline), the autonomous AI coding agent for VS Code, to use Eden AI for access to 500+ models.

## Overview

Cline is a VS Code extension that gives you an autonomous AI agent capable of creating files, running terminal commands, and completing multi-step coding tasks. By pointing it at Eden AI's OpenAI-compatible endpoint, you get:

- **500+ models**: Access OpenAI, Anthropic, Google, Cohere, and more through a single API key
- **Cost savings**: Leverage Eden AI's competitive pricing across all providers
- **Provider flexibility**: Switch models without leaving VS Code

## Prerequisites

- [VS Code](https://code.visualstudio.com) installed
- Eden AI API key from [https://app.edenai.run](https://app.edenai.run)

## Installation

1. Open VS Code
2. Go to Extensions (`Ctrl+Shift+X` / `Cmd+Shift+X`)
3. Search for **"Cline"**
4. Click **Install**

## Configuration

### Step 1: Open Cline Settings

1. Click the **Cline icon** in the VS Code activity bar (or open the Command Palette with `Ctrl+Shift+P` / `Cmd+Shift+P` and run `Cline: Open Settings`)
2. Click the **Settings gear icon** in the top-right corner of the Cline panel

### Step 2: Select the API Provider

In the API Provider dropdown, select **"OpenAI Compatible"**.

### Step 3: Set the Base URL and API Key

Fill in the following fields:

| Field | Value |
|---|---|
| **Base URL** | `https://api.edenai.run/v3/llm` |
| **API Key** | Your Eden AI API key |
| **Model ID** | e.g. `anthropic/claude-sonnet-4-5` |

Click **Save**.

### Alternative: Edit settings.json directly

You can also configure Cline by editing your VS Code `settings.json` (`Ctrl+Shift+P` → `Open User Settings (JSON)`):

<CodeGroup>
```json settings.json
{
  "cline.apiProvider": "openai",
  "cline.openAiBaseUrl": "https://api.edenai.run/v3/llm",
  "cline.openAiApiKey": "YOUR_EDEN_AI_API_KEY",
  "cline.openAiModelId": "anthropic/claude-sonnet-4-5"
}
```
</CodeGroup>

## Features

### Autonomous Task Completion

Cline can autonomously complete multi-step tasks:

1. Open Cline from the activity bar
2. Describe your task: *"Add a login endpoint to this Express app with JWT authentication"*
3. Cline reads relevant files, writes code, and runs commands — asking for approval at each step

### File Editing

Cline proposes file changes as diffs you can review and accept or reject before they are applied.

### Terminal Commands

Cline can run shell commands (with your approval) to install dependencies, run tests, or start a dev server.

### Context Mentions

Reference specific files or folders directly in your prompt using `@`:

- `@filename.ts` — attach a file to the context
- `@problems` — include current VS Code diagnostics
- `@url` — fetch a URL and include its content

## Switching Models Mid-Task

You can change the active model at any time from the Cline settings panel without restarting VS Code. This lets you use a cheaper model for exploration and a more powerful one for final implementation.

## Troubleshooting

### Authentication Errors (401)

1. Confirm your Eden AI API key is correct — copy it from [app.edenai.run](https://app.edenai.run) → **API Keys**
2. Make sure there are no leading/trailing spaces in the key
3. Test the key directly:

```bash
curl -X POST https://api.edenai.run/v3/llm/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "anthropic/claude-sonnet-4-5",
    "messages": [{"role": "user", "content": "ping"}]
  }'
```

### Model Not Found

Ensure you are using the `provider/model` format. For example:

```
# Correct
anthropic/claude-sonnet-4-5

# Incorrect
claude-sonnet-4-5
```

### Connection Issues

1. Verify the base URL is exactly: `https://api.edenai.run/v3/llm`
2. Check Eden AI status: [https://app-edenai.instatus.com](https://app-edenai.instatus.com)

## Best Practices

### Choose the Right Model for the Task

- **Exploration / quick edits**: `anthropic/claude-haiku-4-5` or `openai/gpt-3.5-turbo` — low latency, low cost
- **Standard feature work**: `anthropic/claude-sonnet-4-5` or `openai/gpt-4o` — best balance
- **Complex refactors / architecture**: `anthropic/claude-opus-4-5` or `google/gemini-2.5-pro` — maximum reasoning

### Review Before Approving

Cline will always show you what it plans to do before acting. Review file diffs and terminal commands carefully, especially when working in production codebases.

### Use Context Mentions

Include specific files with `@filename` so Cline has accurate context and does not hallucinate APIs or function signatures that don't exist.
